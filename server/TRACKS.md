# Track workflow

The six demo clips are original project assets generated by
`client/scripts/generate-demo-audio.mjs`. Tracks 001–003 use hand-authored note
sequences. Tracks 004–006 use seeded probabilistic or Markov-style rules. All
six clips and compositions are released under CC0-1.0 by this project.

## Add or edit a track

1. Add a neutral audio filename such as `track-007.wav` under
   `client/public/audio/`. The name must not reveal its classification.
2. Create a migration that inserts or updates `private.tracks`, including the
   title, source type, classification, duration, pack, reveal text, and license.
3. Regenerate `client/src/lib/supabase/database.types.ts` from the production
   schema.
4. Preview the public audio URL, run the database tests, and play one complete
   game before enabling the track.

Authorized administrators can also call `admin_upsert_track` and
`admin_list_tracks`. Authorization requires either a row in `private.admins` or
an Auth `app_metadata.role` value of `admin`; browser-editable user metadata is
never accepted. The service-role key remains server-only and is not needed by
the game client.
